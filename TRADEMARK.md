## What You Can Do

### Use the Code Freely

The WudiBuddy Agents source code is licensed under the Apache License 2.0. You are free to:

- Use, modify, and distribute the code
- Create derivative works
- Use the software for any purpose, including commercial use

### Make Factual Statements

You may make accurate, factual statements about your relationship to the project:

- "Based on WudiBuddy Agents"
- "Built with WudiBuddy Agents technology"
- "Compatible with WudiBuddy Agents"
- "Fork of WudiBuddy Agents"

### Contribute to the Project

Contributors to the official WudiBuddy Agents repository may use the trademarks when discussing their contributions.

## What You Cannot Do

### Use WudiBuddy Branding for Forks

If you create a fork or derivative work, you **must**:

- Choose a different name that does not include "WudiBuddy"
- Remove or replace all WudiBuddy logos and icons
- Update the bundle identifier in a full namespace migration
- Remove WudiBuddy service domains unless connecting to official WudiBuddy services

### Imply Official Endorsement

You may not:

- Use "WudiBuddy" or "WudiBuddy Agents" as your product name
- Use the WudiBuddy logo as your application icon
- Suggest that your fork is the official version
- Imply that Honf or WudiBuddy endorses your product

### Create Confusion

You may not use the trademarks in any way that:

- Suggests your product is created by or affiliated with Honf.
- Could cause confusion between your product and the official WudiBuddy Agents
- Disparages Honf or the WudiBuddy Agents project

## Branding Locations

For those creating forks, the following files contain branding that should be updated:

| File | Contains |
|------|----------|
| `apps/electron/electron-builder.yml` | Product name, bundle ID, copyright |
| `apps/electron/resources/` | Application icons |
| `packages/shared/src/branding.ts` | Service URLs |

## Examples

### Acceptable

- "MyAgent - based on WudiBuddy Agents"

### Not Acceptable
